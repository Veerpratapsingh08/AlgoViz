# AlgoViz Documentation

Welcome to the internal documentation for AlgoViz! This directory contains architectural overviews, technical standards, and user experience reviews to help you understand the core philosophies of the project and make robust open-source contributions.

## Available Documents

- [**ARCHITECTURE.md**](./ARCHITECTURE.md): An overview of the project, its goals, components, and long-term vision.
- [**PROJECT_HYGIENE.md**](./PROJECT_HYGIENE.md): The official guide to our repository structure, naming conventions, CSS standards, and pull request quality gates. **Must read** before submitting major PRs!
- [**OSS_USABILITY_REVIEW.md**](./OSS_USABILITY_REVIEW.md): An analysis of the app's usability from a learner's perspective, highlighting current UI/UX feedback and future priorities for improvement.

## How to use this documentation

When contributing to AlgoViz (as outlined in our root [`CONTRIBUTING.md`](../CONTRIBUTING.md)), please refer to these documents to ensure your feature aligns with our design language, component structure, and educational goals.
